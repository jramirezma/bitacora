El problema

Determine el valor de verdad de la proposición: $(\forall x \in \mathbb{R})(\exists y \in \mathbb{R})\ [2x + 3y = 1]$

¿Qué hay que ver acá?

Compara este ejercicio con el anterior: la única diferencia es que el $\forall$ y el $\exists$ cambiaron de lugar. Antes había que encontrar un único $y$ que funcionara para todos. Ahora es al revés: para cada $x$ que llegue, puedes buscar un $y$ distinto, uno hecho a medida.

El $\forall x$ viene primero: $x$ llega y recién entonces buscas un $y$ que funcione. Como $y$ puede ser diferente para cada $x$, tienes mucha más libertad. No hay un único $y$ que aguante todo: hay uno por cada $x$.

La ecuación $2x + 3y = 1$ despejada en términos de $y$ da $y = \frac{1 - 2x}{3}$. Para cualquier $x$ real que elijas, esa expresión produce un número real. No hay división por cero, no hay raíz de negativo, no hay trampa. Siempre existe ese $y$.

Esta vez el $y$ no tiene que ser el héroe de toda la historia. Solo tiene que resolver el problema de un $x$ a la vez, como un técnico que repara cada aparato por separado en lugar de inventar una solución universal para todos los electrodomésticos del mundo.

Algo que vale la pena saber

Lo que acabas de hacer, despejar $y$ en función de $x$, es exactamente definir una función: $f(x) = \frac{1 - 2x}{3}$. Demostrar que esta proposición es verdadera es lo mismo que demostrar que esa función existe y está bien definida para todo real. Cada vez que defines una función en matemáticas o en código, estás afirmando implícitamente un $(\forall x)(\exists y)$: para cada entrada existe una salida.

La demostración

Desarrollo
Justificación
Sea $x \in \mathbb{R}$ un real cualquiera.
Instanciar el $\forall x$: hay que mostrar que funciona para cualquier $x$.
Tomar $y = \dfrac{1 - 2x}{3}$.
Construir el $y$ explícitamente. Puede ser distinto para cada $x$.
$y = \dfrac{1 - 2x}{3} \in \mathbb{R}$
$\mathbb{R}$ es cerrado bajo suma, resta y división por $3 \neq 0$.
$2x + 3y = 2x + 3\cdot\dfrac{1-2x}{3} = 2x + (1 - 2x) = 1$
Verificar que este $y$ satisface la condición.
$2x + 3y = 1$ $\checkmark$
La condición se cumple para el $y$ construido.
$(\forall x \in \mathbb{R})(\exists y \in \mathbb{R})\ [2x + 3y = 1]$ es verdadera.
Como $x$ era cualquier real y siempre existe $y = \frac{1-2x}{3}$, la proposición vale para todos.

Comprobemos con números

Toma $x = 4$: su $y$ es $\frac{1 - 8}{3} = \frac{-7}{3}$. Verifica: $2(4) + 3\cdot\frac{-7}{3} = 8 - 7 = 1$. Perfecto. Ahora prueba con $x = -2$: $y = \frac{1+4}{3} = \frac{5}{3}$. Verifica: $2(-2) + 3\cdot\frac{5}{3} = -4 + 5 = 1$. También perfecto. La fórmula nunca falla porque está construida exactamente para eso.

Resultado. La proposición $(\forall x \in \mathbb{R})(\exists y \in \mathbb{R})\ [2x + 3y = 1]$ es verdadera. Para cada $x \in \mathbb{R}$, el valor $y = \frac{1-2x}{3}$ siempre existe en $\mathbb{R}$ y satisface la ecuación.

¿Ya usé esto sin saberlo?

Cada función que escribes en código es un $(\forall x)(\exists y)$ en acción: para cada entrada válida, la función promete producir una salida. Cuando una función lanza una excepción o devuelve null para ciertos valores, está rompiendo esa promesa: hay un $x$ para el que no existe $y$. Por eso los tipos de retorno en lenguajes como TypeScript o Kotlin te obligan a ser explícito cuando una función puede no devolver nada: te están pidiendo que declares honestamente si tu $(\forall x)(\exists y)$ es verdadero o no.
