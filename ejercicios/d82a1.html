Ejercicio

a) Implementar la clase Bolígrafo con sus atributos y métodos correspondientes.

b) Instanciar 3 objetos de dicha clase y cargar sus datos.

c) Mostrar el valor total de los 3 bolígrafos.

d) Verificar si los 3 bolígrafos son del mismo color.

e) Mostrar el bolígrafo más caro.

Diagrama de clases

Nota de notación: El diagrama sigue la convención UML estándar: nombre : Tipo tanto en atributos como en métodos (ej. − precio : float, + getPrecio() : float). Algunas cátedras usan el orden inverso (Tipo nombre, más cercano a la sintaxis de Java), que es lo que puede exigirse en el examen. Ambas variantes representan lo mismo; la diferencia es solo de formato.

Análisis

El ejercicio tiene dos partes bien diferenciadas: definir la clase y luego operar sobre tres instancias de ella desde el programa principal. La clase en sí es sencilla: cuatro atributos y los métodos mínimos necesarios para que el programa principal pueda consultarlos.

Los incisos c, d y e se resuelven todos en el Begin/End, no dentro de la clase. La clase solo expone getters; la lógica de comparación y suma vive afuera.

Para el inciso d basta comparar en cadena: si a == b y b == c, los tres son iguales (por transitividad). Para el inciso e se necesita encontrar el máximo de tres valores, lo que se resuelve con dos condiciones anidadas: primero verificar si a gana a los otros dos, luego si lo hace b; si ninguna se cumple, gana c.

Tres bolígrafos, una sola tinta: el clásico problema de quién se quedó sin lapicera a mitad del parcial.

Getters
Solo se exponen los atributos que el programa principal necesita consultar: precio (para sumar y comparar) y color (para verificar igualdad). El encapsulamiento se respeta: el exterior no accede directamente a los campos. 
Begin
    a = creaobjeto Boligrafo();

    a.leer(); b.leer(); c.leer();
Inciso b — Instanciación
Se crean tres objetos independientes de la clase Boligrafo y se cargan sus datos. Cada objeto tiene su propio espacio de atributos en memoria. 
    // c. Valor total
    pa = a.getPrecio();
    pb = b.getPrecio();
    pc = c.getPrecio();
    print(pa + pb + pc);
Inciso c — Suma de precios
Se guardan los tres precios en variables auxiliares para no llamar al getter múltiples veces. La suma se imprime directamente. 
    // d. Mismo color
    ca = a.getColor();
    cb = b.getColor();
    cc = c.getColor();
    if (ca == cb) and (cb == cc) then
        print("Son del mismo color");
    else
        print("No son del mismo color");
Inciso d — Comparación de color
Comparar ca == cb y cb == cc es suficiente por transitividad: si a igual a b y b igual a c, entonces los tres son iguales. No hace falta comparar ca == cc. 
    // e. Boligrafo mas caro
    if (pa >= pb) and (pa >= pc) then
        a.mostrar();
    elseif (pb >= pa) and (pb >= pc) then
        b.mostrar();
    else
        c.mostrar();
End
Inciso e — Máximo de tres
Se verifica si a supera a los otros dos; si no, se verifica lo mismo con b. Si ninguno de los dos primeros gana, necesariamente gana c. En caso de empate, se muestra el primero que cumpla la condición.
Implementación en Java
Boligrafo.java 
import java.util.Scanner;

public class Boligrafo {

    // En Java cada atributo lleva su propio modificador private.
    // No existe el bloque private: de C++.
    private String color;
    private String marca;
    private String industria;
    private float precio;

    // leer() recibe un Scanner como parámetro para leer desde consola.
    // En el pseudocódigo ese detalle estaba implícito en read().
    public void leer(Scanner sc) {
        System.out.print("Color: ");
        color = sc.next();
        System.out.print("Marca: ");
        marca = sc.next();
        System.out.print("Industria: ");
        industria = sc.next();
        System.out.print("Precio: ");
        precio = sc.nextFloat();
    }

    public void mostrar() {
        System.out.println(color + " " + marca + " " + industria + " " + precio);
    }

    // En Java el tipo de retorno va antes del nombre del método,
    // al revés de la notación UML estándar (nombre : Tipo).
    public float getPrecio() { return precio; }
    public String getColor() { return color; }
}
Main.java 
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {

        // new Boligrafo() es el equivalente Java de "creaobjeto Boligrafo()".
        // El Scanner se crea una sola vez y se comparte entre los tres objetos.
        Scanner sc = new Scanner(System.in);
        Boligrafo a = new Boligrafo();
        Boligrafo b = new Boligrafo();
        Boligrafo c = new Boligrafo();

        a.leer(sc);
        b.leer(sc);
        c.leer(sc);

        // c. Valor total — en Java las variables locales se declaran con tipo explícito.
        float pa = a.getPrecio();
        float pb = b.getPrecio();
        float pc = c.getPrecio();
        System.out.println("Total: " + (pa + pb + pc));

        // d. Mismo color — los String se comparan con .equals(), nunca con ==.
        // El operador == compara referencias de memoria, no contenido.
        String ca = a.getColor();
        String cb = b.getColor();
        String cc = c.getColor();
        if (ca.equals(cb) && cb.equals(cc))
            System.out.println("Son del mismo color");
        else
            System.out.println("No son del mismo color");

        // e. Bolígrafo más caro — misma lógica que el pseudocódigo.
        if (pa >= pb && pa >= pc)
            a.mostrar();
        else if (pb >= pa && pb >= pc)
            b.mostrar();
        else
            c.mostrar();

        sc.close(); // buena práctica: liberar el recurso de entrada al terminar
    }
}
Resumen. Se implementó la clase Bolígrafo con cuatro atributos (color, marca, industria, precio) y los métodos leer(), mostrar(), getPrecio() y getColor(). Desde el programa principal se instanciaron tres objetos y se resolvieron los tres requerimientos: suma de precios, verificación de color idéntico por transitividad, y búsqueda del más caro mediante comparación de máximo entre tres valores.
