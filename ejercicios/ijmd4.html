El problema

Simplifique la siguiente expresión utilizando las leyes de conjuntos. Indique la ley que utiliza en cada paso: $$[A \cup (B \cup C)] \cup (B \cap C)$$

¿Qué hay que ver acá?

La expresión une dos cosas: el bloque $[A \cup (B \cup C)]$ y el conjunto $B \cap C$. La clave es notar que $B \cap C$ siempre está contenido en $B$, y $B$ ya está dentro de $B \cup C$, que a su vez está dentro de $A \cup (B \cup C)$. Así que $B \cap C$ ya está "adentro" del primer bloque: unirlo no agrega nada nuevo.

La ley de absorción dice que si $X \subseteq Y$, entonces $Y \cup X = Y$. Acá $B \cap C \subseteq B \subseteq B \cup C \subseteq A \cup (B \cup C)$, así que unir $B \cap C$ al primer bloque lo absorbe sin dejar rastro.

El camino más directo es usar la asociatividad de la unión para reagrupar, y luego aplicar absorción. Con tres conjuntos el árbol de operaciones puede tener varias formas, pero la asociatividad garantiza que el resultado es el mismo sin importar cómo se agrupen.

$B \cap C$ intenta unirse a la fiesta, pero resulta que ya estaba invitado desde el principio. La ley de absorción le cierra la puerta amablemente.

Un dato que cambia todo

La asociatividad de la unión de conjuntos, $A \cup (B \cup C) = (A \cup B) \cup C$, tiene un equivalente directo en los sistemas de bases de datos distribuidas. Cuando una consulta une los resultados de múltiples nodos, el orden en que se combinan los conjuntos de resultados no importa: el resultado final es el mismo. Esto permite que el planificador de consultas distribuidas reorganice libremente el árbol de operaciones para minimizar la transferencia de datos entre nodos, eligiendo el agrupamiento que resulte más eficiente sin alterar el resultado.

Vamos por partes

Desarrollo
Justificación
$[A \cup (B \cup C)] \cup (B \cap C)$
Expresión original.
$A \cup [(B \cup C) \cup (B \cap C)]$
Ley asociativa de la unión: reagrupar para aislar $B \cup C$ y $B \cap C$.
$A \cup (B \cup C)$
Ley de absorción: $(B \cup C) \cup (B \cap C) = B \cup C$, porque $B \cap C \subseteq B \subseteq B \cup C$.

Comprobemos con un ejemplo

Toma $A = \{1\}$, $B = \{2, 3\}$, $C = \{3, 4\}$. $B \cup C = \{2,3,4\}$, $B \cap C = \{3\}$. Lado izquierdo: $[A \cup (B \cup C)] \cup (B \cap C) = \{1,2,3,4\} \cup \{3\} = \{1,2,3,4\}$. Resultado simplificado: $A \cup (B \cup C) = \{1\} \cup \{2,3,4\} = \{1,2,3,4\}$. Coinciden ✓.

Resultado. $[A \cup (B \cup C)] \cup (B \cap C) = A \cup B \cup C$

¿Ya usé esto sin saberlo?

La asociatividad de operaciones sobre conjuntos es lo que permite a los motores de bases de datos distribuidas reordenar libremente las operaciones de unión entre tablas. Cuando una consulta combina resultados de múltiples fuentes, el optimizador elige el orden de agrupamiento que minimiza el costo de transferencia de datos, sabiendo que el resultado final no cambia gracias a la asociatividad. Sin esta propiedad, el planificador estaría atado al orden en que el programador escribió la consulta, perdiendo oportunidades críticas de optimización.
