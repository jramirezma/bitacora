El problema

a. Instanciar 2 objetos de la clase Rectangulo.
b. Crear un método que retorne si es cuadrado o no.
c. Crear un método que imprima área y perímetro.
d. Crear un método que retorne un booleano si un rectángulo puede contener a otro.

¿De qué se trata esto?

La idea es modelar un rectángulo como un objeto. Un objeto es simplemente algo que tiene datos propios (ancho y alto) y que sabe hacer cosas con esos datos. Primero diseñamos el molde — la clase — y después lo usamos para fabricar rectángulos concretos.

Todo rectángulo queda completamente descripto con dos valores: ancho y alto. Con eso solo ya podemos responder todas las preguntas del ejercicio.

La clase va a tener cuatro responsabilidades: guardar las medidas, detectar si es cuadrado, mostrar área y perímetro, y compararse con otro rectángulo para saber si lo contiene. Cada responsabilidad es un método distinto.

Un cuadrado es un rectángulo que se esforzó por ser simétrico. No lo discriminemos, merece su propio método.

Manos a la obra

El diagrama muestra el molde completo: los atributos son privados (solo el objeto los conoce) y los métodos son públicos (cualquiera los puede llamar desde afuera).

Rectangulo
- float ancho
- float alto
+ Rectangulo(float ancho, float alto)
+ boolean esCuadrado()
+ void mostrarDimensiones()
+ boolean puedeContener(Rectangulo otro)

Los atributos llevan - porque son privados. Los métodos llevan + porque son la interfaz pública. puedeContener recibe un objeto del mismo tipo, lo que muestra que una clase puede relacionarse consigo misma.

CLASE Rectangulo 
// Constructor
Rectangulo(float ancho, float alto) {
    self.ancho = ancho
    self.alto  = alto
}

// b. ¿Es cuadrado?
boolean esCuadrado() {
    SI self.ancho == self.alto
        retornar verdadero
    SINO
        retornar falso
}

// c. Área y perímetro
void mostrarDimensiones() {
    float area      = self.ancho * self.alto
    float perimetro = 2 * (self.ancho + self.alto)
    IMPRIMIR "Área:      " + area
    IMPRIMIR "Perímetro: " + perimetro
}

// d. ¿Puede contener a otro?
boolean puedeContener(Rectangulo otro) {
    SI self.ancho > otro.ancho Y self.alto > otro.alto
        retornar verdadero
    SINO
        retornar falso
}
  Recibe las medidas al crear el objeto. Guarda el ancho en el atributo propio. Guarda el alto en el atributo propio.       Devuelve verdadero o falso. Cuadrado = todos los lados iguales. Si se cumple, confirmamos. Si no, descartamos.       No devuelve nada (void). Cuántos cuadraditos entran adentro. La longitud del borde completo. Muestra el resultado en pantalla. Ídem para el perímetro.       Recibe el rectángulo a comparar. Las dos condiciones deben cumplirse. Entra: ambas medidas son menores. No entra: al menos una no cumple.  
PROGRAMA PRINCIPAL 
// Instanciar 2 objetos
Rectangulo rect1 = nuevo Rectangulo(10, 10)
Rectangulo rect2 = nuevo Rectangulo(5, 8)

// b. ¿Es cuadrado?
SI rect1.esCuadrado()
    IMPRIMIR "rect1 ES un cuadrado"
SINO
    IMPRIMIR "rect1 NO es un cuadrado"

// c. Área y perímetro
rect1.mostrarDimensiones()
rect2.mostrarDimensiones()

// d. ¿Puede contener al otro?
SI rect1.puedeContener(rect2)
    IMPRIMIR "rect1 puede contener a rect2"
SINO
    IMPRIMIR "rect1 NO puede contener a rect2"
  10×10 → será cuadrado. 5×8 → rectángulo común.     esCuadrado() devuelve boolean. Resultado: ES cuadrado.         Área: 100 | Perímetro: 40 Área: 40 | Perímetro: 26     10 > 5 Y 10 > 8 → verdadero. Resultado: puede contener.    
Resultado. Se modeló la clase Rectangulo con dos atributos privados (ancho y alto) y cuatro métodos públicos: el constructor inicializa el objeto; esCuadrado() compara los dos lados; mostrarDimensiones() imprime área y perímetro; y puedeContener() compara ambas dimensiones contra las de otro rectángulo. Se instanciaron dos objetos y se verificó el comportamiento de cada método.

¿Y esto para qué me sirve?

Modelar figuras geométricas como objetos no es solo un ejercicio académico. En motores de videojuegos y aplicaciones gráficas, los rectángulos son la base de la detección de colisiones: antes de calcular si dos personajes chocaron, el motor pregunta si sus rectángulos delimitadores se superponen. El método puedeContener() de este ejercicio es exactamente esa lógica.

Además, la idea de encapsular datos (ancho y alto privados) y exponer solo comportamiento (los métodos públicos) es el principio detrás de cualquier librería de interfaz gráfica: uno no manipula los píxeles directamente, sino que le dice al objeto "mostrá tu área" o "¿cabés dentro de esta ventana?".
