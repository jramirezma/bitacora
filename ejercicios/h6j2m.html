El problema

Sea $f(x) = ax + b$, y suponga que para todo $x \in \mathbb{R}$ se tiene que $f(f(x)) = bx + a$. Calcule el valor de $a + b$.

¿De qué se trata esto?

Tenemos una función lineal $f(x) = ax + b$ con dos constantes desconocidas. La condición $f(f(x)) = bx + a$ nos dice cómo se ve la función aplicada dos veces seguidas. La idea es calcular $f(f(x))$ explícitamente, y como el resultado tiene que ser igual a $bx + a$ para todo $x$, los coeficientes de ambos lados deben coincidir exactamente.

Dos polinomios son iguales para todo $x$ si y solo si tienen exactamente los mismos coeficientes. Eso convierte una igualdad de funciones en un sistema de ecuaciones con $a$ y $b$.

Este principio se llama identificación de coeficientes: en lugar de resolver una ecuación con incógnitas numéricas, igualamos los coeficientes del mismo término en ambos lados.

La función se aplica a sí misma y el resultado ya está dado. Solo hay que calcular ese resultado y ver qué condiciones impone.

Vamos por partes

Paso 1: calcular f(f(x))

Desarrollo
Justificación
$f(x) = ax + b$
Función dada.
$f(f(x)) = f(ax + b)$
Se sustituye $f(x)$ como argumento de $f$.
$f(ax+b) = a(ax+b) + b$
Se aplica la regla de $f$: multiplicar el argumento por $a$ y sumar $b$.
$f(f(x)) = a^2x + ab + b$
Se distribuye y se agrupan los términos independientes.

Paso 2: identificar coeficientes

Desarrollo
Justificación
$a^2x + ab + b = bx + a$
Se iguala el resultado obtenido con la condición del enunciado.
$a^2 = b$
Coeficientes de $x$: deben coincidir en ambos lados.
$ab + b = a$
Términos independientes: deben coincidir en ambos lados.

Paso 3: resolver el sistema

Desarrollo
Justificación
$b(a + 1) = a$
Se factoriza $b$ en la segunda ecuación.
$a^2(a+1) = a$
Se sustituye $b = a^2$ de la primera ecuación.
$a^3 + a^2 - a = 0$
Se expande y se pasan todos los términos al mismo lado.
$a(a^2 + a - 1) = 0$
Se factoriza $a$.
$a = 0 \qquad$ o $\qquad a = \dfrac{-1 \pm \sqrt{5}}{2}$
Dos casos: $a = 0$ directo, o se aplica la fórmula cuadrática a $a^2 + a - 1 = 0$.

Paso 4: calcular a + b en cada caso

Desarrollo
Justificación
$a = 0 \Rightarrow b = 0^2 = 0$
$a + b = 0$
Caso trivial: $f(x) = 0$, que efectivamente cumple $f(f(x)) = 0 = bx + a$.
$a = \dfrac{-1+\sqrt{5}}{2}$

$(-1+\sqrt{5})^2 = 1 - 2\sqrt{5} + 5 = 6 - 2\sqrt{5}$

$b = a^2 = \dfrac{6-2\sqrt{5}}{4} = \dfrac{3-\sqrt{5}}{2}$
Se eleva $a$ al cuadrado expandiendo $(-1+\sqrt{5})^2$ con $(p+q)^2 = p^2+2pq+q^2$, luego se simplifica dividiendo por 2.
$a + b = \dfrac{-1+\sqrt{5}}{2} + \dfrac{3-\sqrt{5}}{2} = \dfrac{2}{2} = 1$
Se suman. Los $\sqrt{5}$ se cancelan entre sí.
$a = \dfrac{-1-\sqrt{5}}{2}$

$(-1-\sqrt{5})^2 = 1 + 2\sqrt{5} + 5 = 6 + 2\sqrt{5}$

$b = a^2 = \dfrac{6+2\sqrt{5}}{4} = \dfrac{3+\sqrt{5}}{2}$
Mismo procedimiento con la otra raíz. El signo del $\sqrt{5}$ cambia pero el resultado tiene la misma forma.
$a + b = \dfrac{-1-\sqrt{5}}{2} + \dfrac{3+\sqrt{5}}{2} = \dfrac{2}{2} = 1$
Los $\sqrt{5}$ también se cancelan. Mismo resultado que el caso anterior.

Antes de cerrar, verifiquemos

Para el caso $a + b = 1$, tomamos la solución $a = \frac{-1+\sqrt{5}}{2}$, $b = \frac{3-\sqrt{5}}{2}$ y verificamos que $a^2 = b$:

$$a^2 = \left(\frac{-1+\sqrt{5}}{2}\right)^2 = \frac{1 - 2\sqrt{5} + 5}{4} = \frac{6 - 2\sqrt{5}}{4} = \frac{3-\sqrt{5}}{2} = b \checkmark$$

Y verificamos que $ab + b = a$:

$$b(a+1) = \frac{3-\sqrt{5}}{2} \cdot \frac{1+\sqrt{5}}{2} = \frac{3+3\sqrt{5}-\sqrt{5}-5}{4} = \frac{-2+2\sqrt{5}}{4} = \frac{-1+\sqrt{5}}{2} = a \checkmark$$
Resultado. Los valores posibles de $a + b$ son: $$\boxed{a + b = 0} \quad \text{(caso trivial)} \qquad \text{y} \qquad \boxed{a + b = 1} \quad \text{(casos no triviales)}$$

¿Dónde ya existe esto?

Identificar coeficientes es exactamente lo que hace un compilador cuando resuelve un sistema de restricciones de tipos. Cuando el compilador ve que dos expresiones deben ser del mismo tipo para todo valor posible, no compara caso por caso: compara las estructuras, igual que acá comparamos los coeficientes de $x$ y los términos independientes por separado. En lenguajes con inferencia de tipos, como Haskell o Rust, el compilador construye ecuaciones sobre los tipos desconocidos y las resuelve, sin que el programador tenga que escribir los tipos explícitamente. La matemática detrás de esa inferencia es la misma idea que aplicamos en este ejercicio.
