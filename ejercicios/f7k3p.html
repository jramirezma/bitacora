El problema

a) Identificar 5 clases donde cada clase debe tener 5 atributos significativos.
b) Implementar dos clases definidas en el inciso a.
   b.1) Instanciar dos objetos para cada clase.
   b.2) Para los objetos definidos, comparar dos atributos de cada clase a través de un método.
Contexto: Carnavales de Bolivia.

¿De qué se trata esto?

El ejercicio tiene dos partes. Primero hay que pensar: ¿qué "cosas" existen en los carnavales de Bolivia que se puedan modelar como objetos? Una comparsa, un disfraz, un músico, una reina, una entrada folklórica — todas son candidatas. Cada una tiene características propias: eso son los atributos.

Identificar clases es como hacerle una ficha a cada cosa del mundo real que queremos representar en el programa. Los atributos son los campos de esa ficha.

En la segunda parte tomamos dos de esas clases, las implementamos en pseudocódigo, creamos dos objetos de cada una y escribimos un método que compare dos de sus atributos. Comparar puede significar decir cuál es mayor, cuál tiene más, o si son iguales — depende del atributo.

Modelar los carnavales de Oruro en una clase es casi tan difícil como llegar a un asiento en la tribuna.

Inciso b) — Implementación

Se implementan las clases Comparsa y Musico. Para cada una se definen dos objetos y un método que compara dos de sus atributos. En Comparsa se compara la cantidad de integrantes. En Musico se comparan los años de experiencia.

CLASE Comparsa 
// Constructor
Comparsa(String nombre, String danza,
         int cantidadIntegrantes,
         String ciudadOrigen, int añoFundacion) {
    self.nombre               = nombre
    self.danza                = danza
    self.cantidadIntegrantes  = cantidadIntegrantes
    self.ciudadOrigen         = ciudadOrigen
    self.añoFundacion         = añoFundacion
}

// b.2) Compara cantidad de integrantes con otra comparsa
String compararIntegrantes(Comparsa otra) {
    SI self.cantidadIntegrantes > otra.cantidadIntegrantes
        retornar self.nombre + " tiene más integrantes"
    SINO SI self.cantidadIntegrantes < otra.cantidadIntegrantes
        retornar otra.nombre + " tiene más integrantes"
    SINO
        retornar "Ambas comparsas tienen la misma cantidad"
}
  Parámetros en varias líneas por legibilidad.     Guarda el nombre de la comparsa. Tipo de danza que representa. Cuántos bailarines y músicos la forman. De qué ciudad es originaria. Cuándo fue fundada.       Recibe otra Comparsa para comparar. Si esta tiene más, ella gana.   Si la otra tiene más, la otra gana.   Si son iguales, empate.  
CLASE Musico 
// Constructor
Musico(String nombre, String instrumento,
       int añosExperiencia,
       String comparsa, boolean esDirector) {
    self.nombre           = nombre
    self.instrumento      = instrumento
    self.añosExperiencia  = añosExperiencia
    self.comparsa         = comparsa
    self.esDirector       = esDirector
}

// b.2) Compara años de experiencia con otro músico
String compararExperiencia(Musico otro) {
    SI self.añosExperiencia > otro.añosExperiencia
        retornar self.nombre + " tiene más experiencia"
    SINO SI self.añosExperiencia < otro.añosExperiencia
        retornar otro.nombre + " tiene más experiencia"
    SINO
        retornar "Ambos músicos tienen la misma experiencia"
}
  Parámetros en varias líneas por legibilidad.     Nombre del músico. Instrumento que toca. Cuántos años lleva tocando. A qué comparsa pertenece. true si dirige la banda.       Recibe otro Musico para comparar. Si este tiene más años, él gana.   Si el otro tiene más, el otro gana.   Si son iguales, empate.  
PROGRAMA PRINCIPAL 
// b.1) Instanciar dos objetos de Comparsa
Comparsa c1 = nuevo Comparsa(
    "Los Caporales del Sur", "Caporales",
    120, "Oruro", 1998)
Comparsa c2 = nuevo Comparsa(
    "Morenada Central", "Morenada",
    95, "La Paz", 2003)

// b.2) Comparar atributo: cantidadIntegrantes
IMPRIMIR c1.compararIntegrantes(c2)

// b.1) Instanciar dos objetos de Musico
Musico m1 = nuevo Musico(
    "Don Rufino", "Bombo",
    15, "Los Caporales del Sur", verdadero)
Musico m2 = nuevo Musico(
    "Jhonny Quispe", "Trompeta",
    8, "Morenada Central", falso)

// b.2) Comparar atributo: añosExperiencia
IMPRIMIR m1.compararExperiencia(m2)
  Comparsa con 120 integrantes.     Comparsa con 95 integrantes.       Resultado: "Los Caporales del Sur tiene más integrantes"   Director de banda, 15 años.       Músico con 8 años, no es director.       Resultado: "Don Rufino tiene más experiencia"
Resultado. Se identificaron 5 clases del contexto carnavalero boliviano: Comparsa, Musico, Disfraz, ReinaDelCarnaval y EntradaFolklorica, cada una con 5 atributos significativos. Se implementaron Comparsa y Musico: se instanciaron dos objetos de cada clase y se compararon los atributos cantidadIntegrantes y añosExperiencia respectivamente a través del método compararIntegrantes() y compararExperiencia().

¿Y esto para qué me sirve?

Identificar clases a partir del mundo real es la habilidad central del diseño orientado a objetos. En aplicaciones reales — una app de gestión de eventos, un sistema de acreditación para un festival, una plataforma de venta de entradas — el primer paso siempre es exactamente este: ¿qué entidades existen en este dominio y qué datos las describen?

El método de comparación que se implementó acá es la base de funcionalidades como ordenar listas, filtrar resultados o determinar rankings. Cuando una app muestra "las 10 comparsas más grandes del carnaval", por atrás está corriendo exactamente esta lógica: comparar el atributo de un objeto contra el de otro.
