Lo que nos piden

Si $A = \{a,b,c\}$ y $B = \{c,d,e\}$, calcule $\mathcal{P}(A - B)$.

¿De qué se trata esto?

Dos operaciones encadenadas, igual que antes: primero calculas $A - B$, que es lo que queda de $A$ después de quitarle los elementos que también están en $B$. Luego calculas el conjunto potencia de ese resultado.

El conjunto potencia $\mathcal{P}(X)$ es el conjunto de todos los subconjuntos posibles de $X$, incluyendo el conjunto vacío $\emptyset$ y el propio $X$. Si $X$ tiene $n$ elementos, entonces $\mathcal{P}(X)$ tiene exactamente $2^n$ elementos.

Piensa en el conjunto potencia como todas las formas posibles de elegir elementos de $X$: puedes no elegir ninguno, elegir solo algunos, o elegirlos todos. Si $X$ tiene 2 elementos, hay $2^2 = 4$ formas de elegir. Si tiene 3, hay $2^3 = 8$. El número crece rápido.

El conjunto potencia es la versión matemática de ese amigo que cuando le preguntas "¿qué quieres comer?" responde con todas las combinaciones posibles del menú.

Antes de resolver, esto:

El conjunto potencia $\mathcal{P}(X)$ tiene exactamente $2^n$ elementos cuando $X$ tiene $n$ elementos. Ese $2$ no es casualidad: para cada elemento de $X$ hay exactamente dos opciones, incluirlo o no incluirlo en el subconjunto. Multiplicar $2$ por sí mismo $n$ veces da el total de combinaciones. Este mismo razonamiento aparece en computación cada vez que se trabaja con números binarios: un número de $n$ bits puede representar exactamente $2^n$ valores distintos, porque cada bit tiene dos estados posibles, $0$ o $1$.

Justificación
$A - B = \{a,b,c\} - \{c,d,e\}$

$A - B = \{a, b\}$
$c$ está en $B$, así que se va. $a$ y $b$ no están en $B$, así que se quedan.
$|\,A - B\,| = 2 \implies |\,\mathcal{P}(A-B)\,| = 2^2 = 4$
Un conjunto de $n$ elementos tiene $2^n$ subconjuntos.
$\mathcal{P}(A - B) = \{\,\emptyset,\ \{a\},\ \{b\},\ \{a,b\}\,\}$
Listar todos los subconjuntos: el vacío, los unitarios y el total.

Una prueba rápida

Verifica que hay exactamente 4 elementos: $\emptyset$, $\{a\}$, $\{b\}$ y $\{a,b\}$. Cada uno es un subconjunto de $\{a,b\}$: ninguno contiene elementos fuera de $\{a,b\}$, y no falta ninguna combinación posible. Los 4 coinciden con $2^2 = 4$. Perfecto.

Resultado. $\mathcal{P}(A - B) = \{\,\emptyset,\ \{a\},\ \{b\},\ \{a,b\}\,\}$

¿Ya usé esto sin saberlo?

El conjunto potencia aparece directamente en los problemas de optimización combinatoria en informática. Encontrar el mejor subconjunto de elementos que satisfaga cierta condición, como el problema de la mochila, requiere en principio revisar todos los subconjuntos posibles, es decir, recorrer el conjunto potencia completo. Con $n$ elementos hay $2^n$ subconjuntos, lo que hace que estos algoritmos sean exponenciales y se vuelvan impracticables muy rápido. Por eso gran parte de la investigación en algoritmos busca atajos que eviten explorar todo el conjunto potencia.

Preguntas para explorar

  1. El conjunto potencia de un conjunto de n elementos tiene 2^n elementos, la misma cantidad que los valores representables con n bits en un sistema binario. ¿Cómo se relaciona formalmente el conjunto potencia con la representación binaria de números? ¿Puedes describir una biyección entre subconjuntos de {a,b,c} y números de 3 bits?
  2. En el problema de la mochila (knapsack problem), se busca el subconjunto de objetos con mayor valor que quepa en una mochila de capacidad limitada. La solución por fuerza bruta recorre el conjunto potencia completo. ¿Qué técnicas usa la programación dinámica para evitar explorar los 2^n subconjuntos?
  3. En teoría de la computación, el problema de decidir si un subconjunto de números suma exactamente un valor dado (subset sum) es NP-completo. ¿Qué significa que un problema sea NP-completo y por qué el crecimiento exponencial del conjunto potencia tiene algo que ver con esa clasificación?
  4. Las máscaras de bits (bitmasks) en programación representan subconjuntos de un conjunto finito usando números enteros, donde cada bit indica si un elemento está incluido o no. ¿Cómo se implementaría iterar sobre todos los subconjuntos de un conjunto de n elementos usando máscaras de bits? ¿Cuál es la complejidad temporal de ese proceso?
